# SEO export: вечерние платья

Экспорт отдельной страницы услуги:

- URL в проекте: `/services/custom-tailoring/women/dresses/evening`
- Локальная preview-ссылка: `http://localhost:8080/services/custom-tailoring/women/dresses/evening`
- H1: `Индивидуальный пошив вечерних платьев`
- Title: `Пошив вечерних платьев на заказ в Москве — Ателье 15/13`
- Description: `Индивидуальный пошив вечерних платьев на заказ в Москве. Для свадьбы, выпускного или торжества. Премиум-ткани, идеальная посадка. От 80 000 ₽.`
- Цена на странице: `от 80 000 ₽`

## Что внутри

- `index.html` — статический HTML-снимок страницы с относительными путями к стилям и изображениям.
- `page-data.json` — данные конкретной SEO-страницы: title, description, H1, intro, FAQ, материалы, факторы цены, структура.
- `assets/` — CSS/JS, которые нужны для просмотра страницы отдельно.
- `images/` — изображения, используемые на этой странице.
- `source/` — исходные JS-файлы, из которых сейчас генерируется структура страниц услуг.

## Текущая структура страницы

- `h1` — главный заголовок услуги.
- `h2` — основные секции: галерея, почему ателье, стоимость, процесс, материалы, FAQ.
- `h3` — карточки, факторы цены и вопросы FAQ.
- `strong` — ключевые акценты, отмеченные в макете как bold/strong.
- JSON-LD — BreadcrumbList, Service, FAQPage.

## Важно для SEO-правок

Если SEO-специалист предложит правки по текстам, мета-тегам, FAQ или структуре, удобнее переносить их в `seo/serviceMenuData.js`, потому что именно оттуда сервер собирает страницу.
